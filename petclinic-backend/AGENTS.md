# AGENTS.md — Backend (Spring Boot 3.5, Java 21)

## Fast paths
- Run: `./mvnw spring-boot:run`
- All tests: `./mvnw test`
- Single test: `./mvnw test -Dtest=VisitTest`
- API regression suite: `./postman-tests.sh` (requires `node` and `jq`)

## Architecture
- Controller-centric: REST controllers in `.../rest/` inject repositories and mappers directly.
- Example: `OwnerRestController` composes `OwnerRepository`, `PetRepository`, `VisitRepository`, `PetTypeRepository`, plus `OwnerMapper`/`PetMapper`/`VisitMapper`.
- Repositories: thin Spring Data interfaces in `.../repository/`.
- Domain entities in `.../model/`; DTO source files in `.../rest/dto/`.
- MapStruct interfaces in `.../mapper/`; generated implementations go under `target/generated-sources/` only.
- Global REST error handling: `ExceptionControllerAdvice` → returns `ProblemDetail`.

## Contract and code generation
- API contract snapshot: root `openapi.yaml` (not inside backend resources).
- `MyOpenAPIDidNotChangeTest` compares live `/v3/api-docs.yaml` against `../openapi.yaml` — run its disabled helper test to update the snapshot intentionally.
- MapStruct code is generated by annotation processors (`mvn clean install`); no active OpenAPI-generator plugin.
- Any change to the OpenAPI contract must be confirmed by a human before updating the snapshot.

## Runtime conventions
- Default profile: `h2` — schema/data reload on each restart.
- PostgreSQL: activate `postgres` profile + `docker-compose.yml`.
- Security off by default (`petclinic.security.enable=false`); roles: `OWNER_ADMIN`, `VET_ADMIN`, `ADMIN`.
- H2 console: http://localhost:8080/h2-console (`jdbc:h2:mem:petclinic`, user `sa`, no password).

## Guardrails
- When changing endpoints, update both tests and `../openapi.yaml` snapshot.
- Never upgrade libraries or change the Java version without explicit human confirmation.
- Lombok conventions: `@Slf4j`, `@RequiredArgsConstructor`, `@Builder`, `@Getter`/`@Setter` selectively.
- `@Transactional` only when strictly necessary.
- Line length ≤ 120 chars.
