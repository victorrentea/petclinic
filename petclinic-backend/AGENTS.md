# AGENTS.md

Guidance for work touching `petclinic-backend/` or backend-facing contracts.

## Commands

```sh
./mvnw spring-boot:run              # Run backend
./mvnw test                         # Unit tests
./mvnw test -Dtest=ClassName#methodName
./mvnw clean install                # Build + generate code (MapStruct, OpenAPI)
./postman-tests.sh                  # API tests (Newman)
```

## Backend Architecture

### Layered Structure

1. REST Controllers (`src/main/java/.../rest/`) - expose API endpoints
2. Mappers (`mapper/`) - MapStruct entity↔DTO conversion
3. Repository Layer (`repository/`) - Spring Data JPA interfaces, with no service layer
4. Domain Model (`model/`) - JPA entities such as `Owner`, `Pet`, `Vet`, `Visit`, `Specialty`, `PetType`, `User`, and `Role`

### Generated Code and Contracts

- OpenAPI spec lives at `src/main/resources/openapi.yml`.
- OpenAPI DTOs are generated under `target/generated-sources/.../rest/dto/`.
- MapStruct mappers are generated under `target/generated-sources/.../mapper/`.
- The root `openapi.yaml` file is a committed API snapshot kept in sync by `MyOpenAPIDidNotChangeTest`.
- After intentional API changes, update the stored snapshot via `updateStoredOpenApiYaml()` in that disabled test.

### Data Flow

Request → REST Controller → Repository / Mapper → JPA Entity

Response ← REST Controller ← Mapper (Entity→DTO) ← Repository

### Living Architecture Documentation

Backend architecture diagrams live in `docs/`:

- `architecture.puml` - PlantUML component diagram defining allowed package dependencies
- `c4model.dsl` - Structurizr C4 model generated by test; do not edit by hand

Architecture tests in `src/test/java/.../architecture/`:

- `ArchitectureTest` enforces package dependencies against `architecture.puml`
- `C4ModelGenerationTest` regenerates `docs/c4model.dsl` with SystemContext, Containers, Components, RepositoryFocus, and MapperFocus views

## Runtime and Data

### Database

- Default database: H2 in-memory, auto-populated
- H2 console: `http://localhost:8080/h2-console`
  - JDBC URL: `jdbc:h2:mem:petclinic`
  - user: `sa`
  - password: empty
- PostgreSQL profile: `spring.profiles.active=postgres` with `docker-compose --profile postgres up`

### Security

- Security is disabled by default
- Enable with `petclinic.security.enable=true`
- Roles: `OWNER_ADMIN`, `VET_ADMIN`, `ADMIN`
- Default user: `admin` / `admin`

### Production Data Fact

- Production has roughly 10,000 owners, so query changes should avoid full-table scans, cartesian joins, and missing indexes

## Backend-Specific Modules

## Conventions

- Use DTOs for API contracts and MapStruct for entity↔DTO mapping
- Prefer constructor injection with Lombok `@RequiredArgsConstructor`
- Use global exception handling via `@RestControllerAdvice`
- Put `@Validated` on `@RequestBody` where validation is needed
- Use `@Transactional` only when strictly necessary
- Prefer selective Lombok annotations such as `@Slf4j`, `@RequiredArgsConstructor`, `@Builder`, `@Getter`, and `@Setter`
- Keep lines at 120 characters or less
- Target Java 21+ and Spring Boot 3
- Follow TDD: write a failing test first, confirm it fails, then implement
- Keep diffs minimal and remove dead code when it is part of the touched area
- When builder chains contain 3 or more properties, put one property per line

## When to Regenerate

Run `./mvnw clean install` after:

- modifying `openapi.yml`
- adding or changing MapStruct mappers
- any change that requires regenerated sources under `target/generated-sources/`
