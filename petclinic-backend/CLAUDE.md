# Backend — CLAUDE.md

Spring Boot 3.5 REST API (Java 21), rooted at `petclinic-backend/`.

## Commands
```sh
./mvnw spring-boot:run              # Run backend (localhost:8080)
./mvnw test                         # All tests
./mvnw test -Dtest=ClassName#method # Single test
./mvnw clean install                # Build + regenerate MapStruct & OpenAPI code
./postman-tests.sh                  # API tests (Newman)
```

## Architecture

**Layers:**
1. REST Controllers (`rest/`) — expose API endpoints
2. Mappers (`mapper/`) — MapStruct entity↔DTO conversion
3. Repository (`repository/`) — Spring Data JPA interfaces (no service layer!)
4. Domain Model (`model/`) — JPA entities

**Data Flow:**
Request → REST Controller → Repository / Mapper → JPA Entity
Response ← REST Controller ← Mapper (Entity→DTO) ← Repository

**Key Patterns:**
- DTOs hand-written in `rest/dto/`; OpenAPI spec auto-generated from code (SpringDoc, code-first)
- `openapi.yaml` at project root kept in sync by `OpenApiSyncTest` — update by fetching `/v3/api-docs.yaml` from running backend
- MapStruct for entity↔DTO mapping
- Constructor injection (`@RequiredArgsConstructor`)
- Global exception handling in `@RestControllerAdvice`

## Living Architecture Documentation

Diagrams in `docs/` — generated by guardrail tests, do not edit by hand:
- `packages.puml` — allowed package dependencies (hand-written, source of truth)
- `generated/C4.dsl`, `generated/c4views/*.puml` — C4 model (generated by `C4ModelExtractorTest`)
- `generated/DomainModel.puml` — JPA entity diagram (generated by `DomainModelExtractorTest`)

Guardrail tests also keep `db.sql` and `openapi.yaml` in sync with the running code.

## Database

- **Default:** Postgres via `./start-database.sh` (localhost:5432)
- **Tests:** Zonky embedded Postgres (auto-started per test class, random port)
- Flyway migrations in `src/main/resources/db/migration/`

## Security

- Disabled by default (`petclinic.security.enable=true` to enable)
- Roles: `OWNER_ADMIN`, `VET_ADMIN`, `ADMIN` — default user: `admin`/`admin`
- Tests use `@WithMockUser(roles = "OWNER_ADMIN")`

## Code Preferences

- Constructor injection for production; `@Autowired` only in tests
- `@Transactional` only when strictly necessary
- `@Validated` on `@RequestBody`
- Use Lombok: `@Slf4j`, `@RequiredArgsConstructor`, `@Builder`, `@Getter`/`@Setter` selectively
- Line length ≤ 120 chars
- Always use TDD: write a failing test first, confirm it fails, then implement
- Builder chains: one property per line, unless only 2 properties total
- Reuse Spring's existing types before inventing new ones (e.g., `Sort.Direction`)
- Small domain enums belong inside the controller as inner enums, not separate files
- `@ExceptionHandler(MethodArgumentTypeMismatchException.class)` → 400 in `@RestControllerAdvice`
