Mobile Design Approaches

Three ways to make PetClinic mobile-responsive — pick your preferred direction

A

Progressive Enhancement

Add CSS breakpoints to existing components. Replace the Bootstrap navbar with a hamburger menu on small screens. Tables become scrollable. Minimal code changes — you get 80% of the mobile improvement with 20% of the effort.

Pros

  • Fastest to implement
  • No component rewrites
  • Low risk

Cons

  • Scrollable tables are still painful on mobile
  • No role-based mobile UX differentiation
  • Mediocre mobile experience
B ★ Recommended

Mobile-First Redesign of Key Screens

Redesign the priority screens (Visits, Vets) with a mobile-first mindset: bottom tab navigation, card-based layouts instead of tables, role-aware nav (staff vs pet owner). Desktop screens stay mostly unchanged; mobile gets purpose-built layouts.

Pros

  • Great mobile UX where it matters most
  • Role-based views fit naturally
  • Cards scale better than tables

Cons

  • More work than option A
  • Requires new mobile-specific CSS/layout logic
C

Angular Material Migration

Replace Bootstrap 3 with Angular Material, which ships with built-in responsive components (mat-table with column hiding, sidenav, bottom-sheet, etc.). Full design system overhaul — best long-term maintainability but the most disruptive.

Pros

  • Best long-term DX and accessibility
  • Responsive components out of the box
  • Modern design system

Cons

  • Largest scope — full UI framework swap
  • High risk of regressions
  • Long implementation time
