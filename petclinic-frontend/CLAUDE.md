# Frontend conventions (petclinic-frontend/)

Module-scoped guidance for the Angular SPA (Angular 16 + Angular Material 16 + Bootstrap 3).
The harness loads this automatically whenever you touch files under `petclinic-frontend/`.
Patterns are extracted from the existing screens — open one sibling screen before adding a new
one to calibrate. **Match the existing screens' look; do not ship default Material styling.**

## Stack & where the styles come from

- Angular 16, Angular Material 16, Bootstrap **3.3.7**, jQuery + Tether (Bootstrap 3 JS deps).
- Bootstrap CSS is NOT a webpack entry — it ships as `src/assets/css/petclinic.css`
  (the full Bootstrap 3.3.7 build, **re-themed** to the Spring PetClinic dark-green palette),
  imported from `src/styles.css`. `styles.css` also imports Material's `indigo-pink` prebuilt theme.
- `angular.json` loads jQuery/Tether/Bootstrap **JS** (for navbar dropdowns), not Bootstrap CSS.
- The three coexist on every page. **Prefer matching the Bootstrap-themed screens over
  Material defaults** — a raw `mat-stroked-button` or a plain `mat-table` will look foreign.

## The app's visual conventions (inferred from vet-list, pettype-list, visits-page, owner-detail)

Every list/detail screen is wrapped the same way and uses Bootstrap `.table .table-striped`:

```html
<div class="container-fluid">
  <div class="container xd-container">
    <h2>Veterinarians</h2>
    <table class="table table-striped"> ... </table>
    <button class="btn btn-default" (click)="...">Add Vet</button>
  </div>
</div>
```

### Tables / grids — coloured-bordered, striped

The themed Bootstrap rules (from `petclinic.css`) are the source of truth — replicate them
exactly when you build a `mat-table` instead of a Bootstrap table:

- **Header cells**: bottom border `2px solid #34302d` (dark theme colour); transparent/white
  background, bold. (There is no grey `#f5f5f5` header fill in the real theme — don't invent one.)
- **Body cells**: top border `1px solid #34302d`; padding `8px`.
- **Zebra striping**: odd rows `background-color: #f9f9f9` (`.table-striped > tbody > tr:nth-of-type(odd)`).

On a `mat-table` reproduce this in the component CSS, e.g.:

```css
.my-table th.mat-mdc-header-cell { font-weight: 700; border-bottom: 2px solid #34302d; }
.my-table td.mat-mdc-cell        { border-top: 1px solid #34302d; }
.my-table tr.mat-mdc-row:nth-child(odd) { background: #f9f9f9; }
```

### Buttons — dark-green Spring theme, single line

`.btn .btn-default` in this theme is **not** grey Bootstrap default:

- text `#f1f1f1`, background `#34302d`, border `1px solid #6db33f` (Spring green), `border-radius: 0`,
  `padding: 6px 12px`, `font-size: 14px`.
- Base `.btn` already sets `white-space: nowrap`.

When you use a `mat-button`/`mat-stroked-button` instead, theme it to this look — and either way:

- **Button and form labels must never wrap to a second line.** Add `white-space: nowrap` to
  any custom button/label, and let the input (not the label/button) absorb flex shrink.
- Reuse the existing `class="btn btn-default"` for new buttons unless there is a concrete reason
  to go Material; consistency beats novelty here.

### Sortable tables — fixed layout, explicit column widths

Any table with sorting (Material `matSort`, or your own toggle) MUST use
`table-layout: fixed` plus an explicit width per column. Otherwise toggling sort (asc/desc, on
any column) re-measures content and the column boundaries jump left/right on every click.

```css
.owners-table { width: 100%; table-layout: fixed; }
.owners-table .mat-column-name { width: 22%; }   /* one explicit width per column */
```

Widths must come from CSS rules, never from cell content.

## Module structure

Feature folders under `src/app/<feature>/` (`owners/`, `vets/`, `pettypes/`, `visits/`,
`specialties/`, `pets/`), each an Angular `NgModule` with its own `*-routing.module.ts`.
Screens are `*-list`, `*-detail`, `*-add`, `*-edit` components (`.html` + `.css` + `.ts` + `.spec.ts`).
Per-component CSS is scoped; put look-and-feel overrides there, global rules in `styles.css`.

## API types are generated — don't hand-edit

`src/app/generated/api-types.ts` is generated from the backend's `../openapi.yaml`:

```sh
npm run generate:api      # openapi-typescript ../openapi.yaml -o src/app/generated/api-types.ts
```

`openapi.yaml` itself is generated by the backend (`./mvnw test`) from controllers/DTOs.
Backend contract changed → regenerate `openapi.yaml` (backend) → `npm run generate:api` here.
Never edit `api-types.ts` by hand (the header says so).

## List state lives in the URL query params

List screens treat **URL query params as the source of truth** for page/size/sort/filter
(see `owner-list.component.ts`): `ngOnInit` subscribes to `route.queryParamMap`, reads
`page`/`size`/`sort`/`lastName`, and every user action calls `router.navigate([], { queryParams,
queryParamsHandling: 'merge' })` rather than mutating component state directly. This keeps the
list back/forward- and refresh-safe and deep-linkable. **Scale note: ~100k owners — never load
all rows; always paginate + filter server-side via these params.**

## Commands (run from petclinic-frontend/)

```sh
npm start                 # dev server on localhost:4200 (base href /petclinic)
npm run build             # production build
npm test                  # Karma tests
npm run test-headless     # headless Chrome tests
npm run generate:api      # regenerate src/app/generated/api-types.ts from ../openapi.yaml
npm run lint:openapi      # spectral lint of ../openapi.yaml
```

## House style

- 2-space indent for TS/HTML/CSS (Angular default). Honour `.editorconfig` if present.
- Keep button/label text on one line (`white-space: nowrap`); keep tables consistent with the
  Bootstrap-themed screens above. When in doubt, copy a sibling screen.
